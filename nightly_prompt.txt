====================
前提
====================

- すでに夜間作業用のベースブランチにチェックアウト済み
  feature/night-ai
- 作業はローカルのみで行う
- git push は絶対に行わない
- すべての作業は Jira チケット単位でブランチを分ける
- 各チケットごとに commit を作成する

====================
対象チケット
====================

以下の Jira チケットを、番号の昇順で処理する。

1. KAN-24
2. KAN-25
3. KAN-26
4. KAN-12

====================
共通ルール
====================

- 各チケットごとに以下を必ず実行すること
  1. Jira チケット内容を取得
  2. 作業用ブランチを作成
  3. 実装
  4. flutter analyze を実行
  5. エラーがあれば修正
  6. commit を作成
  7. 次のチケットへ進む

- 判断が必要な場合は、既存実装・命名規則・設計意図を優先する
- 不明点があっても質問せず、最も安全で影響範囲の小さい選択を行う
- テストや analyze が通らない状態で commit しない

====================
ブランチ運用ルール
====================

- 各チケットのブランチ名は以下の形式とする

  claude/KAN-XX

  例:
  - claude/KAN-24
  - claude/KAN-25

- 各ブランチは必ず以下から作成する

  feature/night-ai

====================
commit ルール
====================

- 各チケットにつき commit は 1 つ以上可
- commit メッセージは以下の形式とする

  <type>_KAN-XX <日本語要約>

  例:
  🐞bugfix_👾KAN-24 カードのレイアウト修正

- type は以下から選択する
  - 🐞bugfix
  - ♻️refactor
  - ✈️feature
  - ✏️UIfix

====================
作業終了条件
====================

- すべてのチケットについて commit が作成されていること
- git push はしていないこと

====================
最終出力
====================

最後に、以下のみを日本語で簡潔に出力すること。

- 各チケットで行った作業の要約
- 作成したブランチ名
- 各ブランチの commit メッセージ一覧

コード全文や差分は出力しない
