あなたはソフトウェアエンジニアとして自律的に作業してください。
以下の指示はすべて確定事項です。確認質問は行わないでください。

====================
前提
====================

- 夜間作業用のベースブランチに最新化したdevからチェックアウト
  claude/night
- 作業はローカルのみで行う
- git push devやgit push mainは絶対に行わない
- git checkout devやgit checkout mainは絶対に行わない
- すべての作業は Jira チケット単位でブランチを分ける
- 各チケットごとに commit を作成する
- 各チケットの作業が終了したら claude/night にマージする

====================
共通ルール
====================

- 各チケットごとに以下を必ず実行すること
  1. Jira チケット内容を取得
  2. 作業用ブランチを作成
  3. 実装
  4. flutter analyze を実行
  5. エラーがあれば修正
  6. commit を作成
  7. 次のチケットへ進む

- 判断が必要な場合は、既存実装・命名規則・設計意図を優先する
- 不明点があっても質問せず、最も安全で影響範囲の小さい選択を行う
- テストや analyze が通らない状態で commit しない

====================
ブランチ運用ルール
====================

- 各チケットのブランチ名は以下の形式とする

  claude/KAN-XX

  例:
  - claude/KAN-24
  - claude/KAN-25

- 各ブランチは必ず以下から作成する

  claude/night

====================
commit ルール
====================

- 各チケットにつき commit は 1 つ以上可
- commit メッセージは以下の形式とする
- commit メッセージにclaudeの署名などの機能説明以外の文言は不要

  <絵文字><type>_👾KAN-XX <日本語要約>

  例:
  🐞bugfix_👾KAN-24 カードのレイアウト修正

- type は以下から選択する
  - 🐞bugfix
  - ♻️refactor
  - ✈️feature
  - ✏️UIfix

====================
作業終了条件
====================

- すべてのチケットについて commit が作成されていること
- 全ての変更についてclaude/nightにマージができているか確認し、flutter analyzeを実行し挙動を確認
- 上記が完了したらブランチを発行してgit push すること

====================
最終出力
====================

最後に、以下のみを日本語で簡潔に出力すること。

- 各チケットで行った作業の要約
- 作成したブランチ名
- 各ブランチの commit メッセージ一覧

コード全文や差分は出力しない
